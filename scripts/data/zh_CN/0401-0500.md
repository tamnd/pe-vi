## Problem 406: 猜数游戏
我们尝试通过提问来确定整数集合{1, 2, ..., n}中的某个特定数。每次提问时我们说一个数，可能会得到下面三个回答中的一个：
-  “你的猜测比准确数要小。”（同时你需要付出成本a），或者
-  “你的猜测比准确数要大。”（同时你需要付出成本b），或者
-  “你猜对了！”（游戏结束）。

给定n, a和b的值，可以找到<em>最优策略</em>最小化<u>在最差的可能情况下</u>的总成本。

例如，如果n = 5, a = 2，以及b = 3，那么我们第一次提问应该猜“**2**”。

如果2比准确数要大（付出成本b=3），我们就知道“**1**”是准确数（总成本为<span style="color:blue;">**3**</span>）。
如果2比准确数要小（付出成本a=2），我们下一次提问猜“**4**”。
如果4比准确数要大（付出成本b=3），我们确信“**3**”是准确数（总成本为2+3=<span style="color:blue;">**5**</span>）。
如果4比准确数要小（付出成本a=2），我们确信“**5**”是准确数（总成本为2+2=<span style="color:blue;">**4**</span>）。
因此，这个策略的最差情况总成本是<span style="color:red;">**5**</span>。可以证明这也是最小的最差情况总成本。事实上，以上描述的就是在一个给定的n, a和b值下的最优策略。

记C(n, a, b)是在给定n, a和b值下最优策略的最差情况总成本。

如下是其中一些例子：
C(5, 2, 3) = 5
C(500, √2, √3) = 13.22073197...
C(20000, 5, 7) = 82
C(2000000, √5, √7) = 49.63755955...

记F<sub>k</sub>是斐波那契数：F<sub>k</sub> = F<sub>k-1</sub> + F<sub>k-2</sub>，起始条件为F<sub>1</sub> = F<sub>2</sub> = 1。
求∑<sub>1≤k≤30</sub>&nbsp;C(10<sup>12</sup>, √k, √F<sub>k</sub>)，并将答案四舍五入到小数点后八位小数。

## Problem 414: 卡布列克常数
6174是个神奇的数；如果我们用将它的数字按降序排列组成的数减去将它的数字按升序排列组成的数，我们得到7641-1467=6174。
更神奇的是，如果我们从任意四位数出发，重复这个排列和相减的过程，最终我们都会得到6174，或者当每一位数字都相同时得0。 
即使这个数没有4位，我们可以将其补上前导零凑足四位，上述结论同样成立。
例如，我们从数0837开始：
8730-0378=8352
8532-2358=6174

6174被称为**卡布列克常数**。这个排序和相减直至到达0或卡布列克常数的过程称为**卡布列克路径**。

对于其他进制和位数，我们也可以考察相应的卡布列克路径。 
不幸的是，并不能保证在所有情况下都存在卡布列克常数；有时这个路径会最终进入一个循环，或者从不同的数出发最后会到达不同的常数。
然而，可以证明，对于5位数，以及b = 6t+3≠9 进制，存在卡布列克常数。
例如，在15进制下为： (10,4,14,9,5)<sub>15</sub>
在21进制下为： (14,6,20,13,7)<sub>21</sub>

记C<sub>b</sub>是b进制5位数的卡布列克常数。定义函数sb(i)为
-  0，如果i = C<sub>b</sub>，或者i在b进制下5个数字全部相同。
-  否则为在b进制下从这个数出发经过卡布列克路径到达C<sub>b</sub>的迭代次数。

注意sb(i)定义在所有整数i &lt; b<sup>5</sup>上。即使i在b进制下不足5位，我们都可以在开始卡布列克路径之前给它加上前导零补足5位。

定义S(b)是所有0 &lt; i &lt; b<sup>5</sup>的函数sb(i)的和。
例如，S(15) = 5274369 
S(111) = 400668930299

对于2 ≤ k ≤ 300，求S(6k+3)的和。
给出这个数的最后18位作为你的答案。
